# EcoRecover Coding Standards
Version: 1.0

## Goal

Generate production-quality code that is consistent, readable, scalable, and maintainable.

---

# General

Prioritize

Correctness
→ Simplicity
→ Readability
→ Maintainability
→ Performance

Never sacrifice readability for cleverness.

---

# Kotlin

Use

- Kotlin only
- Immutable data classes
- StateFlow
- Coroutines
- Extension functions
- Sealed classes for UI state
- Data classes for models
- Enums only when values are fixed

Avoid

- Global mutable state
- lateinit unless required
- !! operator
- Massive Activities
- Business logic inside Composables
