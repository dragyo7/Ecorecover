# EcoRecover Engineering Operating Manual
Version: 1.0

## Mission

Build EcoRecover as a production-quality Android application for intelligent e-waste recycling using modern software engineering practices.

Primary objectives:

- Excellent UX
- Reliable backend
- Maintainable architecture
- AI-assisted development
- Minimal technical debt
- Fast iteration
- Production readiness

# Architecture Rules

Use

- MVVM
- Repository Pattern
- Clean Architecture
- Single Responsibility
- Dependency Injection
- Feature-first organization

Avoid

- God classes
- Business logic in UI
- Circular dependencies
- Duplicate code
- Hardcoded values

---

# Android Rules

Compose only.

Never generate XML unless explicitly requested.

Prefer

StateFlow

rememberSaveable

Immutable UI State

Material3 components

Adaptive layouts

Accessibility

Dark mode support

Animations only when improving UX.

---

# Backend Rules

Prefer

Async endpoints

Dependency Injection

Service layer

Repository layer

DTO separation

Input validation

Proper HTTP status codes

Never

Mix database logic inside routers.

# Folder Philosophy

Every folder should answer one question.

UI

Displays data.

Domain

Contains business rules.

Data

Fetches and stores data.

Nothing else.

---

# Code Quality

Every function should

Do one thing

Be easy to test

Be easy to understand

Prefer composition over inheritance.

Delete dead code.

Never comment obvious code.

---

# Documentation

Document

Public APIs

Complex algorithms

Architecture decisions

Non-obvious logic

Don't document syntax.

Document intent.

---

# Logging

Log

Failures

Unexpected states

Passwords

Secrets

Personal information

# Security

Never expose

API Keys

JWT Secrets

Passwords

Database credentials

Validate every external input.

Escape user content.

Use least privilege.
