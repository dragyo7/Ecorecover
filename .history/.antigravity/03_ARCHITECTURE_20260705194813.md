# EcoRecover Architecture
Version: 2.0

Status: TARGET ARCHITECTURE

Purpose

This document defines the desired architecture of EcoRecover.

It does NOT describe the current implementation.

It defines how the existing application should evolve.

Always extend the existing architecture.

Never rebuild from scratch.

# API Philosophy

Android communicates ONLY through REST APIs.

Never directly access databases.

Never expose engine internals.

Version every endpoint.

Example

/api/v1/search

/api/v1/quote

/api/v1/orders

/api/v1/track

/api/v1/insights

/api/v1/profile
