# EcoRecover User Flows
Version: 2.0

Status: ACTIVE

Purpose

This document defines every major user journey in EcoRecover.

It specifies how users move through the application.

It does NOT define implementation.

It defines experience.

# Philosophy

Every user flow should

Require minimum effort.

Be understandable without instructions.

Require as few taps as possible.

Never expose backend complexity.

Guide the user naturally.

# UX Rules

Search always accessible.

Booking under one minute.

Tracking always available.

Users always know the next step.

Never leave users at dead ends.

Every flow should end with a clear CTA.

------------------------------------------------------------------------------

# Definition of Done

Every flow

Has loading state.

Has error state.

Has empty state.

Uses backend APIs.

Uses existing engines.

Supports dark mode.

Supports accessibility.

Is testable.

------------------------------------------------------------------------------

# Final Principle

Users should never think about the system.

They should simply feel that selling e-waste is fast, transparent, and effortless.
