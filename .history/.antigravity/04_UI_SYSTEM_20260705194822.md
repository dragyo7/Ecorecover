# EcoRecover UI System
Version: 2.0

Status: ACTIVE

Purpose

This document defines the complete visual language, interaction principles,
component system and UX philosophy of EcoRecover.

It does NOT describe implementation.

It describes the experience.

------------------------------------------------------------------------------

# Design Goal

EcoRecover should feel like a premium consumer application.

The interface should be immediately familiar to anyone who has used

Blinkit

Instamart

Uber

Google Maps

PhonePe

CRED

without copying their branding.

Users should understand the application within 30 seconds.

------------------------------------------------------------------------------

# Core UX Philosophy

Fast

Minimal

Friendly

Trustworthy

Transparent

Responsive

Delightful

Every screen should answer one question:

"What is the user trying to do right now?"

------------------------------------------------------------------------------

# User Emotion

Opening EcoRecover should feel like

"I can sell my old electronics in minutes."

NOT

"I'm using recycling software."

------------------------------------------------------------------------------

# Design Principles

Large touch targets.

Minimal text.

Strong hierarchy.

Cards over tables.

Search first.

Visual over technical.

Rounded corners.

Smooth animations.

Consistent spacing.

No clutter.

One primary CTA per screen.

# Color Philosophy

Blue

------------------------------------------------------------------------------

# Typography

Material 3 typography only.

Readable.

Comfortable.

Avoid excessive bold text.

Never use more than three font weights on a screen.

------------------------------------------------------------------------------

# Shape System

Rounded cards

Rounded search bars

Rounded buttons

Rounded chips

Rounded bottom sheets

Rounded dialogs

Rounded navigation

Consistency is mandatory.

------------------------------------------------------------------------------

# Spacing System

Use consistent spacing.

Never visually crowd components.

Prefer whitespace over borders.

Use rhythm throughout the application.

------------------------------------------------------------------------------

# Elevation

Use subtle elevation.

Avoid heavy shadows.

Cards should separate content.

Not decorate it.

------------------------------------------------------------------------------

# Animations

Animations should communicate.

Not decorate.

Allowed

Fade

Scale

Slide

Crossfade

Shared transitions

Loading shimmer

Skeleton screens

Progress animations

Avoid excessive bouncing.

Avoid distracting animations.

------------------------------------------------------------------------------

# Bottom Navigation

Maximum four items.

Home

Orders

Insights

Profile

Never exceed four primary tabs.

------------------------------------------------------------------------------

# UX Laws

Search should always be one tap away.

Booking should take under one minute.

Users should never be confused about the next step.

Every important action should provide feedback.

Every quote should be transparent.

Every pickup should be trackable.

Every interaction should feel responsive.

The interface should feel like a polished consumer product that users trust with valuable electronics—not a college demonstration or engineering dashboard.
